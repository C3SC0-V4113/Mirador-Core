# ADRs

Las decisiones de arquitectura de `mirador-core` viven en esta carpeta.

## Reglas

- Usar la plantilla `template.md`.
- Estados permitidos: `Proposed`, `Accepted`, `Rejected`, `Deprecated`,
  `Superseded`.
- No reescribir la historia de un ADR aceptado. Crear uno nuevo que lo superseda.
- Mantener cada ADR enfocado en una decision.
